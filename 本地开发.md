# 本地开发指南(内部定制版)

基于开源 [deepwiki-open](https://github.com/AsyncFuncAI/deepwiki-open) 的内部部署:
对接自建 GitLab(`git.ymdd.tech`)、使用**阿里云百炼 DashScope**(生成 + embedding,无需本地大模型)、
中文界面 + VSCode 主题,并新增「仓库目录」功能。

## 前置条件

- Docker(含 `docker compose` v2)
- Node 20+(仅前端热更开发需要)
- 一个**阿里云百炼 DashScope** API Key(https://bailian.console.aliyun.com)
- 一个 **GitLab 访问令牌**(scope:`read_api`、`read_repository`)

## 配置:`.env`(项目根目录,已被 git 忽略)

```bash
PORT=8001

# 生成 + embedding 都走 DashScope 的 OpenAI 兼容端点(一个 key 搞定)
OPENAI_API_KEY=<你的 DashScope key>
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DEEPWIKI_EMBEDDER_TYPE=openai

# 仓库目录列表 + 私有库克隆 token 回退
GITLAB_URL=https://git.ymdd.tech
GITLAB_TOKEN=<你的 GitLab token>
```

模型与语言在配置文件里(无需改 `.env`):
- 生成:`api/config/generator.json` → `openai.default_model = qwen-plus`(可改 `deepseek-v3`)
- embedding:`api/config/embedder.json` → `text-embedding-v3`(dim 1024)
- 默认语言:`api/config/lang.json` → `zh`

## 启动方式

> 后端镜像为**纯后端**(:8001);前端是 `web/` 下的 **Nuxt 4** 应用,在本机 `npm run dev`(:3001)运行。

### 1. 后端(Docker)
```bash
docker compose up -d        # 后端 API → http://localhost:8001
```
首次构建几分钟(已把中国区所需补丁烤进镜像)。

### 2. 前端(Nuxt 4,位于 `web/`,HMR 热更新)
```bash
cd web
npm install                 # 一次性
npm run dev                 # → http://localhost:3001
# 需要后端 env(GitLab token 等)时:set -a; . ../.env; set +a; npm run dev
```
- 开发地址 **http://localhost:3001**;改任何 `web/` 下的 `.vue` / `.ts` / `.css` 保存即生效。
- Nitro 会把 `/api/*` 等请求代理到 `:8001` 的后端(见 `web/nuxt.config.ts` 的 `routeRules` 与 `server/api/`),**无需额外配置**。

## 什么时候需要重建镜像

| 改动 | 操作 |
|---|---|
| 前端 `web/**`(`.vue`/`.ts`/`.css`) | 不用重建后端镜像,`cd web && npm run dev` 热更即可 |
| 后端 `api/*.py` | 快速:`docker cp api/<file>.py deepwiki-open-deepwiki-1:/app/api/<file>.py && docker restart deepwiki-open-deepwiki-1`;或 `docker compose build && up -d` |
| `.env` / `docker-compose.yml` | `docker compose up -d`(重建容器以加载新环境变量) |

## 前端生产部署

后端镜像已是**纯后端**(不再内置前端)。前端 `web/` 是标准 Nuxt 4 应用,自行构建部署:
```bash
cd web
npm run build                          # 产物在 web/.output
node .output/server/index.mjs          # 或交给 PM2 / 容器托管
# 通过 SERVER_BASE_URL 指向后端(默认 http://localhost:8001)
```

## 浏览器使用要点

- 首页平台选择器要手动选 **GitLab**(`git.ymdd.tech` 域名不含 "gitlab",不会被自动识别)。
- 「项目列表」在首页左侧标签页:可浏览全部 GitLab 项目、按介绍搜索、一键生成/查看。
- 嵌套组(>2 层路径)受 deepwiki 两段式路由限制,暂不支持浏览。

## 本仓库相对上游 deepwiki-open 的改动(rebase 时参考)

> 注:前端已整体重写为 **Nuxt 4**(`web/`)。以下 `src/*` 条目是历史记录(旧 React 版),对应逻辑均已在 `web/` 中重新实现;迁移说明见 `web/README.md`。

2. **DashScope**:`.env` + `api/config/generator.json`(default_provider=openai)+ `api/config/embedder.json`(text-embedding-v3)。DeepSeek 直连**无 embedding 接口**,故用 DashScope。

6. **WebSocket**:修 `serverBaseUrl` 转 ws/wss 的拼接 bug(page.tsx / slides / workshop)。
7. **Mermaid**:`src/components/Mermaid.tsx` 渲染前清洗 LLM 追加在代码块内的 `Sources:` 行。
